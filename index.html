New trap door function

Focused on intractability of factoring

Vulnerable to General Number Field Sieve (NFS)


p = someprime(2 ** 2048)
q = someprime(2 ** 2048)
n = p * q # composite modulus
phi = (p - 1) * (q - 1) # phi of a prime p is p - 1
public_key = 65537 # small prime for fast encryption/verification
private_key = mod_inv(public_key, phi) # modular inverse, very big number
encryption = lambda m: pow(m, public_key, n)
decryption = lambda c: pow(c, private_key, n)

Elgamal (1985)

Extension of diffie-hellman

Relies on DHP

Uses a key tuple(x, k) where k is ephemeral

Uses a subgroup q of the prime field p


p = 809 # prime field
q = 101 # subgroup prime order, factor of p - 1
g = randrange(2, q) # generator of subgroup
x = randrange(1, q) # private key
y = pow(g, x, p) # public key
def enc(m):
    k = randrange(2, q) # private ephemeral key
    c1 = pow(g, k, p) # public ephemeral key
    c2 = m * pow(y, k, p)
    return (c1, c2)
def dec(ct):
    c1, c2 = ct
    return c2 // pow(c1, x, p)

Mathematical Groups

Collection of numbers upon which a binary operation is defined

Operation must satisfy:

  • Group closed under operation
  • Associativity
  • Identity element (0 or 1)
  • Every element has an inverse
  • Groups can be commutative (abelian)

ECElgamal


curve = curves["secp256k1"] # public
G = curve.g # public
order = curve.n # public
x = randrange(1, order) # private key
Q = x * G # public key
def enc(m):
    M = translateToCurve(m, curve) # place message on curve
    k = randrange(1, order) # private ephemeral key
    C1 = k * G # public ephemeral key
    C2 = M + k * Q # encypted message
    return (C1, C2) # ciphertext pairing
def dec(ct):
    C1, C2 = ct
    return C2 - x * C1 # subtract keys from ciphertext to get M

ECDSA


curve = curves["secp256k1"] # public
order = curve.n
G = curve.g
private_key = randrange(1, n)
public_key = private_key * G
private_ephemeral_key = randrange(1, n)
r = (private_ephemeral_key * G).x % order
h = hash(document)
s = mod_inv(private_ephemeral_key, n) * (h + r * private_key)
signature = (r, s)
def verify(r, s, document)
    u1 = mod_inv(s, n) * hash(document)
    u2 = mod_inv(s, n) * r
    P = u1 * G + u2 * public_key
    return r == P.x % n

Long Term Security

At 256 bits, one would need a Dyson sphere occupying space around a supernova to get enough energy to test every possible key

"...brute-force attacks against 256-bit keys will be infeasible until computers are built from something other than matter and occupy something other than space."

-Bruce Schneier

Achieving 256 bits of Security

  • RSA: ~15000 bits
  • ECC: ~521 bits

Security Against Quantum Computing

Number of Qubits required for Shor's algorithm is 2n for RSA and 6n for ECC

where n is the modulus for RSA, and n is the order of the group on the curve for ECC

Ironically, RSA will be more secure in a Quantum future since n is much larger in RSA
